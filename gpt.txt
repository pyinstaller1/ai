


챗GPT

오픈AI에서 개발한 대화형 인공지능 서비스이다.
회원가입 후, 무료 혹은 유료 구독제로 이용할 수 있으며, 채팅을 하듯이 챗봇에 질문을 입력하는 것만으로 AI를 이용할 수 있다.
여기서 입력하는 대화문을 '프롬프트(Prompt)'라고 하며, 이에 대한 AI의 답변 생성을 두고 '응답(Response)'이 생성된다고 표현한다. 
고객 서비스, 교육, 헬프데스크, 개인 비서, 상담 등 여러 분야에 적용된다.
자연어 처리로 사람의 언어를 이해하고 분석해 적절한 응답을 생성한다.
텍스트 생성, 번역, 요약, 코딩 지원 기능을 제공하며 기사 작성, 문서 요약, 언어 번역, 프로그래밍 코드 작성 및 디버깅을 수행할 수 있다.
GPT(Generative Pre-trained Transformer) 기술 기반으로 사전 학습된 대규모 언어 모델이 입력된 텍스트의 패턴을 분석해 적절한 응답을 생성한다.
대량의 텍스트 데이터를 학습하여 문맥을 이해하고 방대한 양의 데이터를 통해 문장의 흐름과 의미를 파악해 자연스러운 문장을 생성한다.
자동화된 응답 시스템, 학생의 학습 도우미, 마케팅 콘텐츠 제작 등에 활용된다.
오픈AI는 15만개 이상의 H100 GPU를 구매해서 GPT를 개발, 운영하고 있다.

GPT-3.5 Turbo: 빠르고 정확한 답변으로 세상을 놀라게 한 GPT 시리즈의 걸작 (2023년 3월 출시)
GPT-4: 느리고 GPU를 많이 소모하지만 긴 문장을 잘 처리 (2023년 3월 출시)
GPT-4o mini: GPT-4의 GPU 소모 문제를 해결하면서, 비슷한 성능 (2024년 5월 출시)
GPT-4o(Omni): 텍스트 이외에 오디오, 이미지, 비디오 입력도 처리 가능 (2024년 5월 출시)

무료 버전으로는 GPT-4o mini 모델만 이용할 수 있으며, 유료 버전은 GPT-4o, GPT-4o mini를 이용할 수 있다.




딥시크
중국의 량원평 CEO가 창립한 딥시크社가 2024년 12월 발표된 DeepSeek-V3 모델은 서구권 LLM 모델과 경쟁할 수 있을 정도의 성능을 보여 많은 주목을 받았다.
특히 V3를 개발하는 데 있어 고작 80억 원 수준에 불과한 적은 비용이 들었다는 사실이 알려지면서,
수천억 원의 비용을 들여 엄청난 연산 자원을 축적해야만 고성능 AI를 개발할 수 있다고 굳게 믿어왔던 시장의 고정관념을 깨뜨리는 계기가 되었다.
이를 가능하게 한 기술은 전문가 시스템이다.
한 거대한 Al가 모든 것을 다 알도록 만드는 대신, 여러 전문가 AI를 두고, 분야별로 각자 처리하게 하는 기술이다.
수학, 과학, 법률 등의 전문가 AI들이 자신의 분야만 처리한다.
기존 모델에서 1.8조 개의 파라미터가 항상 활성화된다면, DeepSeek에서는 6,710억 개의 파라미터 중 단지 370억 개만 활성화된다.
기존에 GPU 100,000대로 실시한 학습 연산을 2,000대로 처리할 수 있다.
기본적인 추론 알고리즘은 챗GPT, 라마와 같은 트랜스포머 알고리즘이다.
다른 중국 제품 들과 마찬가지로 개인정보 유출 우려가 있으므로 사용하지 않는 것이 좋다.
정부 부처마다 외부 접속이 가능한 PC에서 딥시크 접속을 차단하고 있는 가운데, 금융회사들도 민감한 업무 정보나 개인정보가 유출될 가능성을 막기 위해 차단하고 있다.




크롤링(crawling)
웹 페이지의 HTML 코드를 가져와서 데이터를 추출해 내는 방법이다.
html = request.get(url) 과 같은 파이썬 코드를 실행하면 html 변수에 html 코드가 저장되고 그 안에는 웹 화면에 있던 유용한 정보도 포함되는데 이를 추출하는 것이다.
일반 사용자가 크롬 주소창에 url 을 입력하면 서버에 request 처리가 되고 서버는 <html> 태그를 사용자의 크롬에 response로 보낸다.
크롬은 이 <html> 태그를 해석해서 화면을 그려낸다.
반면 크롤링의 경우 request.get(url) 명령을 API 형식으로 보내고 서버에게서 <html> 을 response로 제공받는데, 크롬처럼 그리지 않고, 그 html 코드를 뒤져서 원하는 정보만 추출하는 것이다.
https://website.com/robots.txt 와 같은 형식으로 robot.txt 에 Disallow: / 와 같은 명령어를 추가하면 외부에서 크롤링되지 않는다.
서버는 받은 request 크롬에서 보낸 것인지, API 크롤링 요청인지 판별할 수 있다.
AI 학습 데이터 수집에 있어서 크롤링은 매우 중요하다.
DB 권한이 없어서 접근 못하는 여러 사이트의 데이터를 반복적인 크롤링으로 가져올 수 있다.
따라서 크롤링 송신 기관과 수신 기간 간의 보안 접근 권한을 조절할 필요가 있다.







RPA(Robotic Process Automation)

반복 작업을 소프트웨어 로봇에게 맡겨서 자동 수행하는 기술이다.
데이터 입력, 이메일 발송, 데이터 전송, 보고서 작성 등 반복적인 업무를 자동화할 수 있다.
RPA는 금융거래 처리, IT 시스템 모니터링, 주문 처리, 재고 관리 등 거의 모든 산업에서 적용할 수 있다.
파이썬에서 pyautogui.click(700, 800) 명령어는 모니터의 xy 좌표 (700, 800) 지점을 마우스 클릭하게 하고,
pyautogui.press('enter') 명령어는 엔터키를 누르게 한다.
반복적인 사무 작업이나 야간 작업을 자동으로 처리하는데 있어서 유용하다.
AI가 결정한 명령을 RPA로 자동 실행한다면 많은 업무를 자동화시킬 수 있다.



온프레미스(On-premise)
기업이나 조직이 자체적으로 소프트웨어, 서버, 하드웨어, 네트워크 등을 관리하고 운영하는 방식이다.
클라우드 기반 서비스와 반대되는 개념으로, 모든 인프라와 데이터를 회사 내부의 시설에서 직접 관리하고 운영하는 형태이다.
물리적인 서버와 인프라는 모두 조직의 자체 데이터 센터나 서버실에 배치된다.
데이터와 인프라를 자체 관리하므로, 보안이 중요한 기업에서 선호된다.
외부 클라우드 서비스에 의존하지 않고 자체적으로 보안을 강화할 수 있지만, 보안 관리와 대응에 필요한 리소스와 전문성이 요구된다.
초기 설치 비용이나 유지 관리 비용이 상대적으로 높을 수 있다.
시스템 운영과 관리가 조직 내부에서 이루어지기 때문에 인프라 관리, 서버 유지보수, 데이터 백업 등을 내부 팀이 처리해야 합다.
